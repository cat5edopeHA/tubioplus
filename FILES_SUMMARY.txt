  - GET /:config/stream/:type/:id.json
  - Returns quality options as url streams
  - Respects user quality preference
  - Builds stream URLs with encrypted config

play.js (206 lines)
  - GET /play/:videoId.mp4?q=QUALITY
  - CRITICAL: Actual video streaming endpoint
  - Validates video ID
  - Runs yt-dlp fresh to get current CDN URLs
  - For 360p: redirects to YouTube's muxed format
  - For 720p+: spawns FFmpeg to mux h264 video + AAC audio
  - Pipes output to response with proper headers
  - Handles client disconnect

API ENDPOINTS:
==============

GET /manifest.json
  - Stremio addon manifest

GET /configure
  - Configuration web UI

POST /configure (implicit via client form)
  - Config form submission

GET /:config/catalog/:type/:id.json
  - Catalog: trending, search, subscriptions, history, watchlater
  - Returns: { metas: [...] }

GET /:config/meta/:type/:id.json
  - Video metadata for a single video
  - Returns: { id, name, poster, description, ... }

GET /:config/stream/:type/:id.json
  - Available quality options for a video
  - Returns: { streams: [{url, name, description}, ...] }

GET /:config/play/:videoId.mp4?q=QUALITY
  - Actual video streaming endpoint
  - Returns: HTTP 200 with video/mp4 stream

GET /health
  - Health check endpoint

ENVIRONMENT VARIABLES:
======================

PORT=8000
  Default: 8000
  Purpose: Server port

ENCRYPTION_KEY=0123456789abcdef...
  Required: 64 hex characters (32 bytes for AES-256)
  Purpose: Config encryption/decryption
  Generate: openssl rand -hex 32

NODE_ENV=production
  Default: development
  Purpose: Production optimizations

HOST=yt.m2bw.net (optional)
  Default: auto-detect from request
  Purpose: Override host in stream URLs for reverse proxy
